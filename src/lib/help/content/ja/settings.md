# 設定タブ
デバイス設定、信号処理、埋め込みパラメータ、キャリブレーション、ショートカット、ログを設定します。

## ペアリング済みデバイス
アプリが検出したすべてのBCIデバイスを一覧表示します。優先デバイス（自動接続先）の設定、デバイスの削除、新しいデバイスのスキャンが可能です。最近検出されたデバイスにはRSSI信号強度が表示されます。

## 信号処理
リアルタイムEEGフィルターチェーンを設定します: ローパスカットオフ（高周波ノイズを除去）、ハイパスカットオフ（DCドリフトを除去）、電源線ノッチフィルター（50または60 Hzの商用電源ハムと高調波を除去）。変更は波形表示とバンドパワーに即座に反映されます。

## EEG埋め込み
連続する5秒間の埋め込みエポック間のオーバーラップを調整します。オーバーラップが高いほど1分あたりの埋め込み数が増え（検索の時間分解能が細かくなる）、ストレージと計算コストが増加します。

## キャリブレーション
キャリブレーションタスクを設定します: アクションラベル（例：「開眼」、「閉眼」）、フェーズの持続時間、繰り返し回数、アプリ起動時にキャリブレーションを自動開始するかどうか。

## キャリブレーション音声ガイダンス（TTS）
キャリブレーション中、アプリはオンデバイスの英語テキスト読み上げを使用して各フェーズを音声で案内します。エンジンはKittenTTS（tract-onnx、約30 MB）をespeak-ng音素化とともに使用しています。モデルは初回起動時にHuggingFace Hubからダウンロードされ、ローカルにキャッシュされます — それ以降はデータがデバイスの外に出ることはありません。音声は以下のタイミングで発話されます: セッション開始、各アクションフェーズ、各休憩（「Break. Next: …」）、セッション完了。espeak-ngがPATH上に必要です（brew / apt / apk install espeak-ng）。英語のみ。

## グローバルショートカット
任意のアプリケーションからラベル、検索、設定、キャリブレーションウィンドウを開くためのシステム全体のキーボードショートカットを設定します。標準のアクセラレータ形式を使用します（例：CmdOrCtrl+Shift+L）。

## デバッグログ
{dataDir}/logs/の日次ログファイルへのサブシステムごとのログを切り替えます。サブシステムにはembedder、devices、websocket、csv、filter、bandsが含まれます。

## アップデート
アプリのアップデートを確認してインストールします。Ed25519署名検証付きのTauri組み込みアップデーターを使用しています。

## 外観
カラーモード（システム / ライト / ダーク）の選択、強いボーダーとテキストのためのハイコントラストの有効化、EEG波形とバンドパワー表示用のチャートカラースキームの選択ができます。色覚バリアフリーパレットも利用可能です。ロケールピッカーから言語も変更できます。

## 目標
1日の記録目標を分単位で設定します。ストリーミング中はダッシュボードにプログレスバーが表示され、目標達成時に通知が送信されます。過去30日間のチャートでは、目標達成日（緑）、半分達成日（アンバー）、いくらか進捗のあった日（薄い色）、未達成日（なし）が表示されます。

## テキスト埋め込み
ラベルと検索クエリはnomic-embed-text-v1.5（約130 MBのONNXモデル、768次元）を使用して埋め込まれます。モデルはHuggingFace Hubから一度だけダウンロードされ、ローカルにキャッシュされます。テキスト類似検索とプロアクティブフックで使用されるセマンティックラベルインデックスの両方を支えています。

## ショートカット
ラベル、検索、設定、キャリブレーションウィンドウを開くためのグローバルキーボードショートカット（システム全体のホットキー）を設定します。すべてのアプリ内ショートカット（⌘Kでコマンドパレット、?でショートカットオーバーレイ、⌘↵でラベル送信）も表示されます。ショートカットは標準のアクセラレータ形式を使用します — 例：CmdOrCtrl+Shift+L。

# アクティビティトラッキング
NeuroSkillはオプションで前面にあるアプリとキーボード・マウスの最終使用時刻を記録できます。両機能はデフォルトオフのオプトインで、完全にローカル、設定 → アクティビティトラッキングで個別に設定可能です。

## アクティブウィンドウトラッキング
バックグラウンドスレッドが毎秒起動し、現在前面にあるアプリケーションをOSに問い合わせます。アプリ名またはウィンドウタイトルが変わると、activity.sqliteに1行挿入されます: アプリケーション表示名（例：「Safari」）、アプリバンドルまたは実行ファイルのフルパス、最前面のウィンドウタイトル（例：ドキュメント名や現在のWebページ）、そのウィンドウがアクティブになったUnix秒タイムスタンプ。同じウィンドウにとどまっている場合は新しい行は書き込まれません — 単一アプリでのアイドル時間はデータベースアクティビティを生成しません。macOSではトラッカーはosascriptを呼び出します。アプリ名とパスにはアクセシビリティ権限は不要ですが、サンドボックスアプリではウィンドウタイトルが空の場合があります。Linuxではxdotoolとxpropを使用します（X11セッションが必要）。WindowsではPowerShellのGetForegroundWindow呼び出しを使用します。

## キーボードとマウスのアクティビティトラッキング
グローバル入力フック（rdev）がシステム全体のすべてのキー押下とマウスまたはトラックパッドイベントをリッスンします。入力内容、押したキー、カーソルの移動先は記録しません — メモリ内の2つのUnix秒タイムスタンプのみを更新します: 最新のキーボードイベント用と最新のマウス/トラックパッドイベント用。これらは60秒ごとにactivity.sqliteにフラッシュされますが、前回のフラッシュ以降に少なくとも1つの値が変更された場合のみです。アイドル期間は痕跡を残しません。設定パネルはライブ更新イベント（最大毎秒1回にスロットル）を受信するため、「最終キーボード」と「最終マウス」フィールドはほぼリアルタイムでアクティビティを反映します。

## データの保存場所
すべてのアクティビティデータは単一のSQLiteファイル~/.skill/activity.sqliteに保存されています。送信、同期、またはいかなる分析にも含まれることはありません。2つのテーブルが管理されています: active_windows（ウィンドウフォーカス変更ごとに1行、アプリ名、パス、タイトル、タイムスタンプ付き）とinput_activity（アクティビティ検出時の60秒フラッシュごとに1行、最終キーボードと最終マウスのタイムスタンプ付き）。両テーブルにはタイムスタンプ列の降順インデックスがあります。WALジャーナルモードが有効なため、バックグラウンド書き込みが読み取りをブロックしません。任意のSQLiteブラウザでいつでもファイルを開いて検査、エクスポート、削除できます。

## 必要なOS権限
macOS — アクティブウィンドウトラッキング（アプリ名とパス）に特別な権限は不要です。キーボードとマウスのトラッキングはCGEventTapを使用するためアクセシビリティアクセスが必要です: システム設定 → プライバシーとセキュリティ → アクセシビリティを開き、リストでNeuroSkillを見つけてオンにします。この権限がなくても入力フックはサイレントに失敗します — タイムスタンプがゼロのままになるだけで、アプリの残りは完全に影響を受けません。権限プロンプトを防ぐには設定 → アクティビティトラッキングでトグルを無効にしてください。Linux — 両機能にはX11セッションが必要です。アクティブウィンドウトラッキングはxdotoolとxpropを使用し、ほとんどのデスクトップディストリビューションにプリインストールされています。入力トラッキングはlibxtstのXRecord拡張を使用します。いずれかのツールが見つからない場合、その機能は警告をログに記録して自身を無効にします。Windows — 特別な権限は不要です。アクティブウィンドウトラッキングはPowerShell経由でGetForegroundWindowを使用し、入力トラッキングはSetWindowsHookExを使用します。

## 無効化とデータの削除
設定 → アクティビティトラッキングの両方のトグルは即座に反映されます — 再起動は不要です。アクティブウィンドウトラッキングを無効にするとactive_windowsへの新しい行の挿入が停止し、メモリ内の現在のウィンドウ状態がクリアされます。入力トラッキングを無効にするとrdevコールバックによるタイムスタンプ更新が停止し、input_activityへの今後のフラッシュが防止されます。既存の行は自動的に削除されません。収集された履歴をすべて削除するには: アプリを終了し、~/.skill/activity.sqliteを削除してから再起動してください。次回起動時に空のデータベースが自動的に作成されます。

# UMAP

## UMAP
セッション比較で使用される3D UMAP投影のパラメータを制御します: 近傍数（ローカル構造 vs. グローバル構造を制御）、最小距離（ポイントのクラスタリング密度）、メトリック（コサインまたはユークリッド）。近傍数が多いほどグローバルトポロジーが保持されます。少ないほど細かいローカルクラスターが明らかになります。投影はバックグラウンドジョブで実行され、結果はキャッシュされます。

# EEGモデルタブ
EEG埋め込みエンコーダーとHNSWベクトルインデックスのステータスを監視します。

## エンコーダーステータス
EEG埋め込みエンコーダーの読み込み状態、アーキテクチャの概要（次元数、レイヤー数、ヘッド数）、.safetensorsウェイトファイルへのパスを表示します。エンコーダーはGPUを使用してオンデバイスで完全に実行されます。

## 本日の埋め込み数
本日のHNSWインデックスに埋め込まれた5秒間のEEGエポック数のライブカウンターです。各埋め込みはその瞬間の神経シグネチャーを捉えるコンパクトなベクトルです。

## HNSWパラメータ
M（ノードあたりの接続数）とef_construction（構築時の検索幅）は最近傍インデックスの品質/速度のトレードオフを制御します。高い値はより良いリコールを提供しますが、メモリ使用量が増加します。デフォルト（M=16、ef=200）はバランスの良い設定です。

## データ正規化
エンコード前の生EEGに適用されるdata_normスケーリングファクターです。デフォルト（10）はMuse 2 / Muse Sヘッドセット向けに調整されています。

# OpenBCIボード
OpenBCIの任意のボード — Ganglion、Cyton、Cyton+Daisy、WiFi Shieldバリアント、Galea — を単独または別のBCIデバイスと併用して接続・設定します。

## ボード選択
使用するOpenBCIボードを選択します。Ganglion（4チャネル、BLE）が最もポータブルなオプションです。Cyton（8チャネル、USBシリアル）はチャネル数が多くなります。Cyton+Daisyでこれが16チャネルに倍増します。WiFi Shieldバリアントはにより1 kHzのWi-Fiストリームに置き換えます。Galea（24チャネル、UDP）は高密度の研究用ボードです。すべてのバリアントが単独または別のBCIデバイスと併用で動作します。

## Ganglion BLE
GanglionはBluetooth Low Energyで接続します。接続をクリックすると、NeuroSkill™は設定されたスキャンタイムアウトまで最も近くでアドバタイズしているGanglionをスキャンします。ボードを3–5 m以内に保ち、電源を入れてください（青色LEDが点滅）。Bluetoothアダプターごとに1つのGanglionのみアクティブにできます。ボードのアドバタイズが遅い場合は設定でBLEスキャンタイムアウトを延長してください。

## シリアルポート（Cyton / Cyton+Daisy）
CytonボードはUSBラジオドングル経由で通信します。シリアルポートフィールドを空白のままにすると最初の利用可能なポートを自動検出します。明示的に入力することもできます（macOSでは/dev/cu.usbserial-…、Linuxでは/dev/ttyUSB0、WindowsではCOM3）。接続をクリックする前にドングルを接続し、シリアルポートの権限を確認してください — Linuxではユーザーをdialoutグループに追加します。

## WiFi Shield
OpenBCI WiFi Shieldは独自の2.4 GHzアクセスポイント（SSID: OpenBCI-XXXX）を作成します。コンピューターをそのネットワークに接続し、IPを192.168.4.1（Shieldのデフォルトゲートウェイ）に設定します。あるいは、Shieldをホームネットワークに参加させることもできます — その場合は割り当てられたIPを入力します。IPフィールドを空白のままにするとmDNS経由で自動検出を試みます。WiFi Shieldは1 kHzでストリーミングします — 信号処理設定でローパスフィルターのカットオフを500 Hz以下に設定してください。

## Galea
Galeaは24チャネルの研究グレードバイオシグナルヘッドセット（EEG + EMG + AUX）で、UDP経由でストリーミングします。GaleaデバイスのIPアドレスを入力するか、ローカルネットワーク上の任意の送信者からのパケットを受信する場合は空白のままにします。チャネル1–8はEEGでリアルタイム分析を駆動します。チャネル9–16はEMG、17–24はAUXです。24チャネルすべてがCSVに保存されます。

## チャネルラベルとプリセット
各物理チャネルに標準10-20電極名を割り当てることで、バンドパワーメトリクス、前頭アルファ非対称性、電極ビジュアライゼーションが電極認識になります。プリセット（前頭、運動、後頭、完全10-20）を使用してラベルを自動入力するか、カスタム名を入力します。最初の4つ以降のチャネルはCSVにのみ記録され、リアルタイム分析パイプラインには使用されません。
