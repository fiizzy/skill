# 개요
NeuroSkill에는 데이터를 클라우드로 전송하지 않고 개인 OpenAI 호환 AI 어시스턴트를 제공하는 선택적 로컬 LLM 서버가 포함되어 있습니다.

## LLM 기능이란 무엇인가요?
LLM 기능은 llama.cpp 기반 추론 서버를 앱 내부에 직접 내장합니다. 활성화하면 WebSocket API와 동일한 로컬 포트에서 OpenAI 호환 엔드포인트(/v1/chat/completions, /v1/completions, /v1/embeddings, /v1/models, /health)를 제공합니다. Chatbot UI, Continue, Open Interpreter 또는 자체 스크립트 등 OpenAI 호환 클라이언트를 연결할 수 있습니다.

## 개인 정보 보호 및 오프라인 사용
모든 추론은 사용자의 기기에서 실행됩니다. 토큰, 프롬프트, 완성 결과가 localhost를 떠나지 않습니다. 유일한 네트워크 활동은 HuggingFace Hub에서의 초기 모델 다운로드입니다. 모델이 로컬에 캐시되면 인터넷 연결을 완전히 해제할 수 있습니다.

## OpenAI 호환 API
서버는 OpenAI API와 동일한 프로토콜을 사용합니다. base_url 매개변수를 허용하는 모든 라이브러리(openai-python, openai-node, LangChain, LlamaIndex 등)가 바로 작동합니다. base_url을 http://localhost:<port>/v1로 설정하고 추론 설정에서 API 키를 구성하지 않았다면 비워두세요.

# 모델 관리
내장 카탈로그에서 GGUF 양자화 언어 모델을 찾아보고, 다운로드하고, 활성화합니다.

## 모델 카탈로그
카탈로그에는 모델 패밀리별로 여러 양자화 변형이 포함된 큐레이션된 모델 패밀리(예: Qwen, Llama, Gemma, Phi)가 나열됩니다. 패밀리 드롭다운으로 찾아본 후 다운로드할 특정 양자화를 선택하세요. ★ 표시된 모델은 해당 패밀리의 추천 기본값입니다.

## 양자화 수준
각 모델은 여러 GGUF 양자화 수준(Q4_K_M, Q5_K_M, Q6_K, Q8_0 등)으로 제공됩니다. 낮은 양자화는 더 작고 빠르지만 일부 품질이 희생됩니다. Q4_K_M이 일반적으로 최적의 균형입니다. Q8_0은 거의 무손실이지만 약 두 배의 메모리가 필요합니다. BF16/F16/F32는 양자화되지 않은 참조 가중치입니다.

## 하드웨어 적합성 배지
각 양자화 행에는 하드웨어 적합도를 추정하는 색상 코드 배지가 표시됩니다: 🟢 훌륭히 실행됨 — GPU VRAM에 여유 있게 들어감. 🟡 잘 실행됨 — VRAM에 빠듯하게 들어감. 🟠 빠듯함 — 부분적 CPU 오프로드 또는 컨텍스트 크기 축소가 필요할 수 있음. 🔴 맞지 않음 — 사용 가능한 메모리에 비해 너무 큼. 추정치는 GPU VRAM, 시스템 RAM, 모델 크기, 컨텍스트 오버헤드를 고려합니다.

## 비전 / 멀티모달 모델
Vision 또는 Multimodal 태그가 지정된 패밀리에는 선택적 멀티모달 프로젝터(mmproj) 파일이 포함됩니다. 채팅 창에서 이미지 입력을 활성화하려면 텍스트 모델과 프로젝터를 모두 다운로드하세요. 프로젝터는 텍스트 모델을 확장합니다 — 독립 모델이 아닙니다.

## 다운로드 및 삭제
'다운로드'를 클릭하면 HuggingFace Hub에서 모델을 가져옵니다. 진행률 표시줄이 실시간 다운로드 상태를 보여줍니다. 언제든지 취소할 수 있습니다. 다운로드된 모델은 로컬에 저장되며 디스크 공간을 확보하기 위해 삭제할 수 있습니다. 모델 디렉토리를 수동으로 수정한 경우 '캐시 새로고침' 버튼으로 카탈로그를 다시 스캔하세요.

# 추론 설정
서버가 모델을 로드하고 실행하는 방식을 미세 조정합니다.

## GPU 레이어
GPU에 오프로드되는 트랜스포머 레이어 수를 제어합니다. 모델이 VRAM에 들어가면 최대 속도를 위해 '전체'로 설정합니다. CPU 전용 추론의 경우 0으로 설정합니다. 중간 값은 모델을 GPU와 CPU에 분할합니다 — 모델이 VRAM 용량을 약간 초과할 때 유용합니다.

## 추론 디바이스(CPU / GPU)
LLM 추론을 CPU에서 실행할지 GPU에서 실행할지 선택합니다. 디바이스 전환은 즉시 적용됩니다 — 서버가 이미 실행 중이면 자동으로 재시작됩니다. CPU에서는 모든 트랜스포머 레이어가 호스트 프로세서에서 실행됩니다(병렬 BLAS에 libomp 사용). GPU에서는 레이어가 Metal(macOS) 또는 Vulkan(Linux/Windows)으로 오프로드됩니다. GPU VRAM이 부족하거나 GPU를 EEG 임베딩용으로 예약할 때 CPU 모드를 사용하세요.

## 컨텍스트 크기
토큰 단위의 KV 캐시 크기입니다. '자동'은 모델의 크기와 양자화를 기반으로 GPU/RAM에 맞는 가장 큰 컨텍스트를 선택합니다. 큰 컨텍스트는 모델이 더 많은 대화 기록을 기억할 수 있지만 더 많은 메모리를 소비합니다. 옵션은 모델의 훈련된 최대값으로 제한됩니다. 메모리 부족 오류가 발생하면 컨텍스트 크기를 줄여보세요.

## 병렬 요청
최대 동시 디코드 루프 수입니다. 높은 값은 여러 클라이언트가 서버를 공유할 수 있게 하지만 최대 메모리 사용량이 증가합니다. 대부분의 단일 사용자 설정에서는 1이 적합합니다.

## API 키
모든 /v1/* 요청에 필요한 선택적 Bearer 토큰입니다. localhost에서 오픈 액세스하려면 비워두세요. 로컬 네트워크에서 포트를 노출하고 액세스를 제한하려면 키를 설정하세요.

# 내장 도구
LLM 채팅은 로컬 도구를 호출하여 정보를 수집하거나 사용자를 대신해 동작을 수행할 수 있습니다.

## 도구 작동 방식
도구 사용이 활성화되면 모델이 대화 중에 하나 이상의 도구 호출을 요청할 수 있습니다. 앱이 도구를 로컬에서 실행하고 결과를 모델에 피드백하여 실제 정보를 응답에 반영할 수 있도록 합니다. 도구는 모델이 명시적으로 요청할 때만 호출됩니다 — 백그라운드에서 실행되지 않습니다.

## 안전한 도구
Date, Location, Web Search, Web Fetch, Read File은 시스템을 수정할 수 없는 읽기 전용 도구입니다. Date는 현재 로컬 날짜와 시간을 반환합니다. Location은 IP 기반 대략적 위치를 제공합니다. Web Search는 DuckDuckGo 즉석 답변 쿼리를 실행합니다. Web Fetch는 공개 URL의 텍스트 본문을 가져옵니다. Read File은 선택적 페이지네이션으로 로컬 파일을 읽습니다.

## 권한이 필요한 도구 (⚠️)
Bash, Write File, Edit File은 시스템을 수정할 수 있습니다. Bash는 앱과 동일한 권한으로 쉘 명령을 실행합니다. Write File은 디스크에 파일을 생성하거나 덮어씁니다. Edit File은 찾기 및 바꾸기 편집을 수행합니다. 기본적으로 비활성화되어 있으며 경고 배지가 표시됩니다. 위험을 이해한 경우에만 활성화하세요.

## 실행 모드 및 제한
병렬 모드는 모델이 여러 도구를 동시에 호출할 수 있습니다(더 빠름). 순차 모드는 하나씩 실행합니다(부작용이 있는 도구에 더 안전). '최대 라운드'는 메시지당 허용되는 도구 호출 / 도구 결과 왕복 횟수를 제한합니다. '라운드당 최대 호출 수'는 동시 도구 호출 수를 제한합니다.

# 채팅 및 로그
모델과 상호작용하고 서버 활동을 모니터링합니다.

## 채팅 창
LLM 서버 카드 또는 트레이 메뉴에서 채팅 창을 엽니다. 마크다운 렌더링, 코드 하이라이팅, 도구 호출 시각화가 포함된 친숙한 채팅 인터페이스를 제공합니다. 대화는 임시적이며 디스크에 저장되지 않습니다. 비전 지원 모델은 드래그 앤 드롭 또는 첨부 버튼을 통해 이미지 첨부를 허용합니다.

## 외부 클라이언트 사용
서버가 OpenAI 호환이므로 모든 외부 채팅 프론트엔드를 사용할 수 있습니다. http://localhost:<port>/v1을 가리키고, API 키를 구성한 경우 설정하고, /v1/models에서 모델 이름을 선택합니다. 인기 옵션: Open WebUI, Chatbot UI, Continue(VS Code), 스크립팅용 curl / httpie.

## 서버 로그
LLM 설정 패널 하단의 로그 뷰어는 서버 출력을 실시간으로 스트리밍합니다. 모델 로딩 진행률, 토큰 생성 속도, 오류를 표시합니다. 상세한 llama.cpp 진단 출력을 위해 고급 섹션에서 '상세' 모드를 활성화하세요. 로그는 자동 스크롤되지만 위로 스크롤하면 일시정지할 수 있습니다.
