# 설정 탭
디바이스 환경 설정, 신호 처리, 임베딩 매개변수, 캘리브레이션, 단축키, 로깅을 구성합니다.

## 페어링된 디바이스
앱이 인식한 모든 BCI 디바이스를 나열합니다. 선호 디바이스(자동 연결 대상)를 설정하거나, 디바이스를 삭제하거나, 새 디바이스를 스캔할 수 있습니다. 최근 감지된 디바이스의 RSSI 신호 강도가 표시됩니다.

## 신호 처리
실시간 EEG 필터 체인을 구성합니다: 저역 통과 차단(고주파 노이즈 제거), 고역 통과 차단(DC 드리프트 제거), 전원선 노치 필터(50 또는 60 Hz 전원 잡음 및 고조파 제거). 변경 사항은 파형 디스플레이와 대역 전력에 즉시 적용됩니다.

## EEG 임베딩
연속 5초 임베딩 에포크 간의 오버랩을 조정합니다. 높은 오버랩은 분당 더 많은 임베딩(검색에서 더 세밀한 시간 해상도)을 의미하지만 저장 공간과 연산량이 증가합니다.

## 캘리브레이션
캘리브레이션 작업을 구성합니다: 동작 라벨(예: "눈 뜨기", "눈 감기"), 단계 지속 시간, 반복 횟수, 앱 실행 시 캘리브레이션 자동 시작 여부.

## 캘리브레이션 음성 안내 (TTS)
캘리브레이션 중에 앱이 기기 내 영어 음성 합성을 사용하여 각 단계를 음성으로 안내합니다. 엔진은 KittenTTS(tract-onnx, ~30 MB)와 espeak-ng 음소화를 기반으로 합니다. 모델은 첫 실행 시 HuggingFace Hub에서 다운로드되어 로컬에 캐시됩니다 — 그 후에는 데이터가 기기를 떠나지 않습니다. 음성은 세션 시작, 각 동작 단계, 모든 휴식("Break. Next: …"), 세션 완료 시 출력됩니다. espeak-ng가 PATH에 필요합니다(brew / apt / apk install espeak-ng). 영어만 지원됩니다.

## 전역 단축키
어떤 애플리케이션에서든 라벨, 검색, 설정, 캘리브레이션 창을 열 수 있는 시스템 전체 키보드 단축키를 설정합니다. 표준 가속기 형식을 사용합니다(예: CmdOrCtrl+Shift+L).

## 디버그 로깅
{dataDir}/logs/의 일일 로그 파일에 대한 하위 시스템별 로깅을 전환합니다. 하위 시스템에는 embedder, devices, websocket, csv, filter, bands가 포함됩니다.

## 업데이트
앱 업데이트를 확인하고 설치합니다. Ed25519 서명 검증이 포함된 Tauri 내장 업데이터를 사용합니다.

## 외관
색상 모드(시스템 / 라이트 / 다크)를 선택하고, 더 강한 테두리와 텍스트를 위한 고대비를 활성화하고, EEG 파형 및 대역 전력 시각화를 위한 차트 색 구성표를 선택합니다. 색약 안전 팔레트를 사용할 수 있습니다. 언어도 로케일 선택기를 통해 여기서 변경합니다.

## 목표
일일 녹화 목표를 분 단위로 설정합니다. 스트리밍 중 대시보드에 진행률 표시줄이 나타나며, 목표 달성 시 알림이 발생합니다. 최근 30일 차트에는 달성(녹색), 절반 달성(주황색), 일부 진행(흐림), 미달성(없음)이 표시됩니다.

## 텍스트 임베딩
라벨과 검색 쿼리는 nomic-embed-text-v1.5(~130 MB ONNX 모델, 768차원)를 사용하여 임베딩됩니다. 모델은 HuggingFace Hub에서 한 번 다운로드되어 로컬에 캐시됩니다. 텍스트 유사도 검색과 프로액티브 훅에서 사용되는 의미적 라벨 인덱스를 모두 구동합니다.

## 단축키
라벨, 검색, 설정, 캘리브레이션 창을 여는 전역 키보드 단축키(시스템 전체 핫키)를 구성합니다. 앱 내 단축키(⌘K — 명령 팔레트, ? — 단축키 오버레이, ⌘↵ — 라벨 제출)도 표시합니다. 단축키는 표준 가속기 형식을 사용합니다 — 예: CmdOrCtrl+Shift+L.

# 활동 추적
NeuroSkill은 선택적으로 전면에 있는 앱과 키보드 및 마우스가 마지막으로 사용된 시간을 기록할 수 있습니다. 두 기능 모두 기본적으로 비활성화된 옵트인이며, 완전히 로컬이고, 설정 → 활동 추적에서 독립적으로 구성할 수 있습니다.

## 활성 창 추적
백그라운드 스레드가 매초 깨어나 OS에 현재 전면에 있는 애플리케이션을 확인합니다. 앱 이름이나 창 제목이 변경되면 activity.sqlite에 한 행이 삽입됩니다: 애플리케이션 표시 이름(예: "Safari"), 애플리케이션 번들 또는 실행 파일의 전체 경로, 최전면 창 제목(예: 문서 이름 또는 현재 웹페이지), 해당 창이 활성화된 Unix 초 타임스탬프. 같은 창에 머물면 새 행이 기록되지 않습니다 — 단일 앱에서의 유휴 시간은 데이터베이스 활동을 생성하지 않습니다. macOS에서 추적기는 osascript를 호출합니다. 앱 이름과 경로에는 접근성 권한이 필요하지 않지만, 샌드박스 앱의 경우 창 제목이 비어 있을 수 있습니다. Linux에서는 xdotool과 xprop을 사용합니다(X11 세션 필요). Windows에서는 PowerShell GetForegroundWindow 호출을 사용합니다.

## 키보드 및 마우스 활동 추적
전역 입력 훅(rdev)이 시스템 전체의 모든 키 누르기와 마우스 또는 트랙패드 이벤트를 수신합니다. 무엇을 입력했는지, 어떤 키를 눌렀는지, 커서가 어디로 이동했는지는 기록하지 않습니다 — 메모리에서 두 개의 Unix 초 타임스탬프만 업데이트합니다: 가장 최근 키보드 이벤트와 가장 최근 마우스/트랙패드 이벤트. 이들은 60초마다 activity.sqlite에 플러시되지만, 마지막 플러시 이후 하나 이상의 값이 변경된 경우에만 기록되므로 유휴 기간에는 흔적이 남지 않습니다. 설정 패널은 실시간 업데이트 이벤트를 수신합니다(최대 초당 1회로 제한되어) "마지막 키보드" 및 "마지막 마우스" 필드에 거의 실시간으로 활동이 반영됩니다.

## 데이터 저장 위치
모든 활동 데이터는 단일 SQLite 파일에 있습니다: ~/.skill/activity.sqlite. 전송, 동기화, 분석에 포함되지 않습니다. 두 개의 테이블이 유지됩니다: active_windows(창 포커스 변경마다 한 행, 앱 이름, 경로, 제목, 타임스탬프 포함)와 input_activity(활동이 감지된 60초 플러시마다 한 행, 마지막 키보드 및 마지막 마우스 타임스탬프 포함). 두 테이블 모두 타임스탬프 열에 내림차순 인덱스가 있습니다. WAL 저널 모드가 활성화되어 백그라운드 쓰기가 읽기를 차단하지 않습니다. 어떤 SQLite 브라우저로든 언제든지 파일을 열고, 검사하고, 내보내거나 삭제할 수 있습니다.

## 필요한 OS 권한
macOS — 활성 창 추적(앱 이름 및 경로)에는 특별한 권한이 필요하지 않습니다. 키보드 및 마우스 추적은 접근성 권한이 필요한 CGEventTap을 사용합니다: 시스템 설정 → 개인 정보 보호 및 보안 → 접근성을 열고 목록에서 NeuroSkill을 찾아 활성화합니다. 이 권한이 없으면 입력 훅이 조용히 실패합니다 — 타임스탬프가 0으로 유지되며 앱의 나머지 부분은 전혀 영향을 받지 않습니다. 설정 → 활동 추적에서 토글을 비활성화하면 권한 프롬프트 자체를 방지할 수 있습니다. Linux — 두 기능 모두 X11 세션이 필요합니다. 활성 창 추적은 대부분의 데스크톱 배포판에 사전 설치된 xdotool과 xprop을 사용합니다. 입력 추적은 libxtst의 XRecord 확장을 사용합니다. 도구가 없으면 해당 기능은 경고를 로깅하고 자체 비활성화됩니다. Windows — 특별한 권한이 필요하지 않습니다. 활성 창 추적은 PowerShell을 통한 GetForegroundWindow를 사용하고, 입력 추적은 SetWindowsHookEx를 사용합니다.

## 비활성화 및 데이터 삭제
설정 → 활동 추적의 두 토글은 즉시 적용됩니다 — 재시작이 필요하지 않습니다. 활성 창 추적을 비활성화하면 active_windows에 새 행 삽입이 중지되고 메모리 내 현재 창 상태가 지워집니다. 입력 추적을 비활성화하면 rdev 콜백이 타임스탬프를 업데이트하는 것을 중지하고 input_activity에 대한 향후 플러시를 방지합니다. 기존 행은 자동으로 제거되지 않습니다. 수집된 모든 기록을 삭제하려면: 앱을 종료하고, ~/.skill/activity.sqlite를 삭제한 후 다시 실행하세요. 다음 시작 시 빈 데이터베이스가 자동으로 생성됩니다.

# UMAP

## UMAP
세션 비교에서 사용되는 3D UMAP 프로젝션의 매개변수를 제어합니다: 이웃 수(로컬 대 글로벌 구조 제어), 최소 거리(점이 얼마나 빽빽하게 클러스터링되는지), 메트릭(코사인 또는 유클리드). 이웃 수가 높을수록 더 많은 글로벌 토폴로지를 보존하고, 낮을수록 세밀한 로컬 클러스터를 드러냅니다. 프로젝션은 백그라운드 작업으로 실행되며 결과가 캐시됩니다.

# EEG 모델 탭
EEG 임베딩 인코더 및 HNSW 벡터 인덱스 상태를 모니터링합니다.

## 인코더 상태
EEG 임베딩 인코더의 로드 여부, 아키텍처 요약(차원, 레이어, 헤드), .safetensors 가중치 파일의 경로를 표시합니다. 인코더는 GPU를 사용하여 완전히 기기 내에서 실행됩니다.

## 오늘의 임베딩
오늘의 HNSW 인덱스에 임베딩된 5초 EEG 에포크 수를 나타내는 실시간 카운터입니다. 각 임베딩은 해당 순간의 신경 서명을 캡처하는 컴팩트 벡터입니다.

## HNSW 매개변수
M(노드당 연결 수)과 ef_construction(구축 중 검색 폭)은 최근접 이웃 인덱스의 품질/속도 트레이드오프를 제어합니다. 높은 값은 더 나은 리콜을 제공하지만 더 많은 메모리를 사용합니다. 기본값(M=16, ef=200)은 좋은 균형입니다.

## 데이터 정규화
인코딩 전 원시 EEG에 적용되는 data_norm 스케일링 팩터입니다. 기본값(10)은 Muse 2 / Muse S 헤드셋에 맞게 조정되어 있습니다.

# OpenBCI 보드
Ganglion, Cyton, Cyton+Daisy, WiFi Shield 변형, Galea 등 모든 OpenBCI 보드를 다른 BCI 디바이스와 함께 또는 단독으로 연결하고 구성합니다.

## 보드 선택
사용할 OpenBCI 보드를 선택합니다. Ganglion(4채널, BLE)이 가장 휴대성이 좋습니다. Cyton(8채널, USB 시리얼)은 더 높은 채널 수를 추가합니다. Cyton+Daisy는 16채널로 두 배가 됩니다. WiFi Shield 변형은 USB/BLE 링크를 1 kHz Wi-Fi 스트림으로 대체합니다. Galea(24채널, UDP)는 고밀도 연구용 보드입니다. 모든 변형은 단독으로 또는 다른 BCI 디바이스와 함께 실행할 수 있습니다.

## Ganglion BLE
Ganglion은 Bluetooth Low Energy를 통해 연결됩니다. 연결을 누르면 NeuroSkill™이 설정된 스캔 타임아웃 동안 가장 가까운 광고 중인 Ganglion을 스캔합니다. 보드를 3–5m 이내에 두고 전원을 켜세요(파란색 LED 깜빡임). Bluetooth 어댑터당 하나의 Ganglion만 활성화할 수 있습니다. 보드의 광고가 느린 경우 설정에서 BLE 스캔 타임아웃을 연장하세요.

## 시리얼 포트 (Cyton / Cyton+Daisy)
Cyton 보드는 USB 라디오 동글을 통해 통신합니다. 시리얼 포트 필드를 비워두면 사용 가능한 첫 번째 포트를 자동 감지하거나, 명시적으로 입력합니다(macOS에서 /dev/cu.usbserial-…, Linux에서 /dev/ttyUSB0, Windows에서 COM3). 연결을 클릭하기 전에 동글을 연결하고 시리얼 포트 권한이 있는지 확인하세요 — Linux에서는 사용자를 dialout 그룹에 추가하세요.

## WiFi Shield
OpenBCI WiFi Shield는 자체 2.4 GHz 액세스 포인트를 생성합니다(SSID: OpenBCI-XXXX). 컴퓨터에서 해당 네트워크에 연결한 후 IP를 192.168.4.1(Shield의 기본 게이트웨이)로 설정합니다. 또는 Shield가 홈 네트워크에 참여할 수 있습니다 — 할당된 IP를 대신 입력하세요. IP 필드를 비워두면 mDNS를 통한 자동 검색을 시도합니다. WiFi Shield는 1 kHz로 스트리밍합니다 — 에일리어싱을 방지하려면 신호 처리 설정에서 저역 통과 필터 차단을 ≤ 500 Hz로 설정하세요.

## Galea
Galea는 EEG + EMG + AUX 센서를 결합한 24채널 연구용 바이오 신호 헤드셋으로 UDP를 통해 스트리밍합니다. Galea 디바이스의 IP 주소를 입력하거나, 로컬 네트워크의 모든 발신자로부터 패킷을 수신하려면 비워둡니다. 채널 1–8은 EEG로 실시간 분석을 구동하고, 채널 9–16은 EMG, 17–24는 AUX입니다. 모든 24채널이 CSV에 저장됩니다.

## 채널 라벨 및 프리셋
각 물리적 채널에 표준 10-20 전극 이름을 할당하여 대역 전력 지표, 전두엽 알파 비대칭, 전극 시각화가 전극을 인식하도록 합니다. 프리셋(전두엽, 운동, 후두엽, 전체 10-20)을 사용하여 라벨을 자동으로 채우거나 사용자 정의 이름을 입력합니다. 첫 4개 이후의 채널은 CSV에만 기록되며 실시간 분석 파이프라인을 구동하지 않습니다.
