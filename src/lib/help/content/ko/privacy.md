# 개인 정보 보호 개요
{app}은 완전한 로컬 우선으로 설계되었습니다. EEG 데이터, 임베딩, 라벨, 설정은 명시적으로 공유를 선택하지 않는 한 기기를 떠나지 않습니다.

# 데이터 저장

## 모든 데이터는 기기에 남습니다
{app}이 기록하는 모든 데이터 — 원시 EEG 샘플(CSV), EEG 임베딩(SQLite + HNSW 인덱스), 텍스트 라벨, 캘리브레이션 타임스탬프, 로그, 설정 — 은 {dataDir}/에 로컬로 저장됩니다. 클라우드 서비스, 서버, 또는 제3자에게 데이터가 업로드되지 않습니다.

## 사용자 계정 없음
{app}은 가입, 로그인, 또는 어떠한 형태의 계정 생성도 요구하지 않습니다. 사용자 식별자, 토큰, 인증 자격 증명이 저장되거나 전송되지 않습니다.

## 데이터 위치
모든 파일은 macOS 및 Linux에서 {dataDir}/ 아래에 저장됩니다. 각 녹화일은 EEG SQLite 데이터베이스와 HNSW 벡터 인덱스가 포함된 자체 YYYYMMDD 하위 디렉토리를 갖습니다. 라벨은 {dataDir}/labels.sqlite에 있습니다. 로그는 {dataDir}/logs/에 있습니다. 이 파일들은 언제든지 삭제할 수 있습니다.

# 네트워크 활동

## 원격 분석 또는 분석 없음
{app}은 사용 분석, 충돌 보고서, 원격 분석, 또는 어떠한 형태의 행동 추적도 수집하지 않습니다. 분석 SDK, 추적 픽셀, 또는 폰홈 비콘이 애플리케이션에 내장되어 있지 않습니다.

## 로컬 전용 WebSocket 서버
{app}은 동반 도구로의 LAN 스트리밍을 위해 로컬 네트워크 인터페이스에 바인딩된 WebSocket 서버를 실행합니다. 이 서버는 인터넷에 노출되지 않습니다. 동일한 로컬 네트워크의 클라이언트에게 파생된 EEG 지표(대역 전력, 점수, 심박수)와 상태 업데이트를 브로드캐스트합니다. 원시 EEG/PPG/IMU 샘플 스트림은 브로드캐스트되지 않습니다.

## mDNS / Bonjour 서비스
{app}은 LAN 클라이언트가 WebSocket 포트를 자동으로 검색할 수 있도록 _skill._tcp.local. mDNS 서비스를 등록합니다. 이 광고는 로컬 전용(멀티캐스트 DNS)이며 네트워크 외부에서는 보이지 않습니다.

## 업데이트 확인
설정에서 '업데이트 확인'을 클릭하면 {app}이 설정된 업데이트 엔드포인트에 접속하여 새 버전을 확인합니다. 이것이 앱이 수행하는 유일한 외부 인터넷 요청이며, 명시적으로 트리거할 때만 발생합니다. 업데이트 번들은 설치 전에 Ed25519 서명으로 검증됩니다.

# Bluetooth 및 디바이스 보안

## Bluetooth Low Energy (BLE)
{app}은 Bluetooth Low Energy 또는 USB 시리얼을 통해 BCI 디바이스와 통신합니다. 연결은 표준 CoreBluetooth(macOS) 또는 BlueZ(Linux) 시스템 스택을 사용합니다. 사용자 정의 Bluetooth 드라이버나 커널 모듈은 설치되지 않습니다.

## OS 수준 권한
Bluetooth 접근에는 명시적 시스템 권한이 필요합니다. macOS에서는 시스템 설정 → 개인 정보 보호 및 보안 → Bluetooth에서 Bluetooth 접근 권한을 부여해야 합니다. {app}은 사용자의 동의 없이 Bluetooth에 접근할 수 없습니다.

## 디바이스 식별자
디바이스 시리얼 번호와 MAC 주소는 BCI 헤드셋에서 수신되어 UI에 표시됩니다. 이러한 식별자는 로컬 설정 파일에만 저장되며 네트워크를 통해 전송되지 않습니다.

# 기기 내 처리

## GPU 추론은 로컬에서 실행
EEG 임베딩 인코더는 wgpu를 통해 로컬 GPU에서 완전히 실행됩니다. 모델 가중치는 로컬 Hugging Face 캐시(~/.cache/huggingface/)에서 로드됩니다. EEG 데이터가 외부 추론 API나 클라우드 GPU로 전송되지 않습니다. 라벨 검색용 텍스트 임베딩은 nomic-embed-text-v1.5를 사용하며 역시 로컬에 캐시됩니다.

## 필터링 및 분석
모든 신호 처리 — overlap-save 필터링, FFT 대역 전력 계산, 스펙트로그램 생성, 신호 품질 모니터링 — 은 로컬 CPU/GPU에서 실행됩니다. 원시 또는 처리된 EEG 데이터가 기기를 떠나지 않습니다.

## 최근접 이웃 검색
유사도 검색에 사용되는 HNSW 벡터 인덱스는 기기에서 완전히 구축되고 쿼리됩니다. 검색 쿼리가 기기를 떠나지 않습니다.

# 사용자의 데이터, 사용자의 통제

## 접근
모든 데이터는 {dataDir}/에 표준 형식(CSV, SQLite, 바이너리 HNSW)으로 저장됩니다. 어떤 도구로든 읽기, 복사, 처리할 수 있습니다.

## 삭제
{dataDir}/ 아래의 파일이나 디렉토리를 언제든지 삭제할 수 있습니다. 걱정할 클라우드 백업이 없습니다. 앱을 제거하면 애플리케이션 바이너리만 제거됩니다 — {dataDir}/의 데이터는 직접 삭제하지 않는 한 그대로 유지됩니다.

## 내보내기
CSV 녹화와 SQLite 데이터베이스는 이식 가능한 표준 형식입니다. 다른 기기로 복사하거나 Python, R, MATLAB 또는 기타 분석 도구로 가져올 수 있습니다.

## 암호화
{app}은 저장 데이터를 암호화하지 않습니다. 디스크 수준 암호화가 필요한 경우 운영 체제의 전체 디스크 암호화(macOS에서 FileVault, Linux에서 LUKS)를 사용하세요.

# 활동 추적

## 활동 추적
활성화하면 NeuroSkill은 현재 전면에 있는 애플리케이션과 키보드 및 마우스가 마지막으로 사용된 시간을 기록합니다. 이 데이터는 ~/.skill/activity.sqlite에 완전히 기기 내에 남습니다 — 서버로 전송되거나, 원격 로깅되거나, 분석에 포함되지 않습니다. 활성 창 추적은 애플리케이션 이름, 실행 파일 경로, 창 제목, 해당 창이 활성화된 Unix 타임스탬프를 캡처합니다. 키보드 및 마우스 추적은 두 개의 타임스탬프(마지막 키보드 이벤트, 마지막 마우스 이벤트)만 캡처합니다 — 키 입력, 입력한 텍스트, 커서 좌표, 클릭 대상은 절대 기록하지 않습니다. 두 기능은 설정 → 활동 추적에서 독립적으로 비활성화할 수 있으며, 기능을 비활성화하면 수집이 즉시 중단됩니다. 기존 행은 자동으로 삭제되지 않지만, 언제든지 activity.sqlite를 삭제하여 제거할 수 있습니다.

## 접근성 권한 (macOS)
macOS에서 키보드 및 마우스 추적은 시스템 수준 입력 이벤트를 가로채는 CGEventTap을 설치하므로 접근성 권한이 필요합니다. Apple은 전역 입력을 읽는 모든 앱에 이 권한을 필수로 요구합니다. 권한은 기능이 활성화될 때만 요청됩니다. 거부하거나 취소하면 훅이 조용히 실패합니다: 앱의 나머지 부분은 정상적으로 계속 작동하며 입력 활동 타임스탬프만 0으로 유지됩니다. 활성 창 추적(앱 이름/경로)은 접근성이 필요하지 않습니다 — 일반 앱 권한 내에서 작동하는 AppleScript/osascript를 사용합니다.

# 요약

## 클라우드 없음
클라우드 없음. 모든 EEG 데이터, 임베딩, 라벨, 설정은 {dataDir}/에 로컬로 저장됩니다.

## 원격 분석 없음
원격 분석 없음. 분석, 충돌 보고서, 사용 추적이 일절 없습니다.

## 계정 없음
계정 없음. 가입, 로그인, 사용자 식별자가 없습니다.

## 하나의 선택적 네트워크 요청
하나의 선택적 네트워크 요청. 업데이트 확인은 명시적으로 트리거할 때만 수행됩니다.

## 완전한 기기 내 처리
완전한 기기 내 처리. GPU 추론, 신호 처리, 검색 모두 로컬에서 실행됩니다.

## 활동 추적은 로컬 전용
활동 추적은 로컬 전용입니다. 창 포커스 및 입력 타임스탬프는 기기의 activity.sqlite에 기록되며 절대 외부로 전송되지 않습니다.
