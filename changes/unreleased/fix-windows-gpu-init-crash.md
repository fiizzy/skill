### Bugfixes

- **Fix Windows Vulkan STATUS_ACCESS_VIOLATION crash**: In debug builds, cubecl/wgpu enables Vulkan validation layers (`InstanceFlags::debugging()`) which trigger `STATUS_ACCESS_VIOLATION` (0xc0000005) on certain Windows GPU drivers during shader compilation. The EEG embed worker and re-embed command now manually initialise the wgpu device with `InstanceFlags::from_build_config().with_env()`, which respects `WGPU_VALIDATION=0` / `WGPU_GPU_BASED_VALIDATION=0` environment variables set at app startup. This keeps Vulkan as the backend while disabling the problematic validation layers in debug builds.
