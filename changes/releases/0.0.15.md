## [0.0.15] ‚Äî 2026-03-11

### Windows support

- **espeak-ng static build ‚Äî Windows MSVC** ‚Äî `scripts/build-espeak-static.ps1`
  builds `espeak-ng.lib` from source using CMake + MSVC on first run, then
  links it statically into the binary; subsequent builds are a no-op.
  Produces a single merged `.lib` (CMake + `lib.exe /OUT`) so that the linker
  sees no duplicate symbol conflicts
- **espeak-ng static build ‚Äî Windows MinGW/GNU** ‚Äî `scripts/build-espeak-static-mingw.sh`
  handles the `*-windows-gnu` target ABI (MSYS2 / cross-compile from
  Linux/macOS); output goes to `espeak-static-mingw/` to avoid collisions
  with the MSVC archive
- **`build.rs` ‚Äî multi-platform espeak dispatch** ‚Äî selects the correct
  build script and archive name based on `CARGO_CFG_TARGET_ENV`:
  `msvc` ‚Üí PowerShell `.ps1`, `gnu` on Windows ‚Üí MinGW `.sh`,
  macOS/Linux ‚Üí Unix `.sh`; links `-lc++` on macOS, `-lstdc++` on Linux/MinGW,
  and omits the C++ flag on MSVC (runtime linked automatically)
- **`build.rs` ‚Äî espeak data copy deferred to release** ‚Äî the espeak-ng data
  directory is no longer copied during `cargo build` / `tauri dev`; the copy
  is skipped in debug builds to break the infinite rebuild loop where
  `build.rs` copies ‚Üí Tauri watcher detects the change ‚Üí `cargo run` ‚Üí repeat.
  An empty placeholder directory is still created so Tauri's resource-path
  validation does not error at startup
- **`fast-hnsw` ‚Äî vendored with Windows fix** ‚Äî `memmap2::Mmap::advise()` and
  `memmap2::Advice` are `#[cfg(unix)]` and not available on Windows; the
  `fast_hnsw` crate's unconditional `advise(Advice::Random)` call caused a
  compile error on the MSVC target; patched locally via
  `src-tauri/vendor/fast-hnsw` with the `advise` call wrapped in
  `#[cfg(unix)]`; vendored until upstream releases a fix
- **`WINDOWS.md`** ‚Äî updated prerequisites: Visual Studio Build Tools 2022
  (**Desktop development with C++** workload, provides `cl.exe`, `lib.exe`,
  Windows SDK) now listed as step 1; CMake doc note updated to cover
  espeak-ng's build system in addition to llama.cpp; added Git as a
  prerequisite for cloning the espeak-ng source; renumbered all steps

### Build / tooling

- **`scripts/tauri-build.js` refactored** ‚Äî now a general Tauri wrapper that
  handles `dev`, `build`, and any other subcommand; non-`dev`/`build`
  subcommands (e.g. `tauri info`, `tauri signer`) pass straight through without
  triggering an espeak pre-build; platform detection now also covers
  `*-windows-gnu` (MinGW)
- **npm `tauri` script** ‚Äî `"tauri": "node scripts/tauri-build.js"` routes all
  `npm run tauri ‚Ä¶` invocations through the wrapper, so `npm run tauri info`,
  `npm run tauri dev`, `npm run tauri build -- --debug`, etc. all work
  consistently across platforms

### LLM ‚Äî WebSocket / REST API

- **WebSocket commands** ‚Äî `llm_status`, `llm_start`, `llm_stop`,
  `llm_catalog`, `llm_download`, `llm_cancel_download`, `llm_delete`,
  `llm_logs` added to the WebSocket command dispatcher (all behind the `llm`
  Cargo feature flag)
- **REST endpoints** ‚Äî matching HTTP shortcuts added to the axum router in
  `api.rs`:
  - `GET  /llm/status` ‚Äî running state, active model name, context size, vision flag
  - `POST /llm/start` ‚Äî load the active model and start the inference server
  - `POST /llm/stop` ‚Äî stop the inference server and free GPU/CPU resources
  - `GET  /llm/catalog` ‚Äî model catalog with per-entry download states
  - `POST /llm/download` ‚Äî start a background model download `{ "filename": "‚Ä¶" }`
  - `POST /llm/cancel_download` ‚Äî cancel an in-progress download
  - `POST /llm/delete` ‚Äî delete a locally-cached model file
  - `GET  /llm/logs` ‚Äî last 500 LLM server log lines
  - `POST /llm/chat` ‚Äî non-streaming chat completion; body: `{ message, images?, system?, temperature?, max_tokens? }`; returns `{ text, finish_reason, tokens }`
- **`LlmServerState::chat()`** ‚Äî new method on the server-state actor handle;
  submits a generate request to the actor's channel and returns an
  `UnboundedReceiver<InferToken>` for streaming; returns `Err` immediately if
  the model is still loading or the actor has exited
- **`extract_images_from_messages()`** ‚Äî helper that decodes all
  `data:<mime>;base64,‚Ä¶` data-URL image parts from an OpenAI-style messages
  array into raw `Vec<u8>` bytes; plain `https://‚Ä¶` URLs are silently skipped;
  call before passing messages to the actor so it receives pre-decoded bytes

### LLM ‚Äî verbose logging

- **`LlmConfig.verbose`** (`bool`, default `false`) ‚Äî when `false` (default),
  all internal llama.cpp / ggml and clip logs are silenced; set `true` to
  see raw tensor-load progress and other low-level detail
- **`mtmd_log_set` silence** ‚Äî `clip_model_loader` uses a separate logger
  (`mtmd_log_set`) that is not affected by `llama_log_set`; the clip logger is
  now silenced via a no-op `extern "C"` callback when `verbose = false`,
  eliminating the tensor-load spam when loading a multimodal projector

### CLI (`cli.ts`)

- **`llm` subcommand group** added:
  - `llm status` ‚Äî print LLM server status (stopped / loading / running)
  - `llm start` ‚Äî load the active model and start the inference server
  - `llm stop` ‚Äî stop the inference server, free GPU memory
  - `llm catalog` ‚Äî list all catalog models with download states
  - `llm download <filename>` ‚Äî start a background model download
  - `llm cancel <filename>` ‚Äî cancel an in-progress download
  - `llm delete <filename>` ‚Äî delete a cached model file
  - `llm logs` ‚Äî print the last 500 LLM server log lines
  - `llm chat` ‚Äî interactive multi-turn chat REPL (WebSocket streaming)
  - `llm chat "message"` ‚Äî single-shot: send one message and stream the reply
  - `llm chat "message" --image a.jpg --image b.png` ‚Äî vision: attach images
    (files are base64-encoded and embedded as `image_url` content parts;
    requires a vision-capable model with mmproj loaded)
- **`--image <path>`** ‚Äî new flag (repeatable) for attaching image files to
  `llm chat` turns
- **`--system <prompt>`** ‚Äî system prompt prepended as a `{ role: "system" }`
  message for `llm chat`
- **`--max-tokens <n>`** ‚Äî maximum tokens to generate per turn
- **`--temperature <f>`** ‚Äî sampling temperature (0 = deterministic, 1 = creative)

### Help UI

### Internals
